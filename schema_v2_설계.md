# 스키마 v2 설계 — 캐릭터 검증 실험 구조

> 목적: 스키마를 **실험 설계** 관점으로 재구성한다.
> 확정 전 설계 문서. 승인되면 이걸 기준으로 `schema.md`·폴더·템플릿·스크립트를 재작성한다.
> 작성: 2026-07-03

---

## 1. 목표 (한 줄)

**매력적인 남캐를 대량 생성하고, 어떤 캐릭터가 반응이 좋은지 검증해서 IP 후보를 뽑는다.**
드라마(에피소드)는 주인공이 아니라, **캐릭터를 검증하기 위한 실험 도구**다.

## 2. 실험 프레임

| 실험 요소 | 스키마 대응 | 성격 |
|---|---|---|
| 통제변수 (고정) | 세계관, 여주 | 항상 같게 유지 → 캐릭터만 비교 가능 |
| 실험 변수 | 남캐 (character) | 대량 생성해서 바꿔가며 테스트 |
| 시행 (trial) | 에피소드 3~5개 | 각 남캐를 여러 상황에서 노출 |
| 측정 | 성과 (performance) | 조회수·ER·저장률 집계 |
| 결과 | IP 후보 선정 | 성과 상위 캐릭터를 IP 후보로 |

## 3. 파이프라인

```
세계관 (고정, 1회 생성)
   │
   ▼
남캐 생성 (여러 개)
   │
   ▼
에피소드 3~5개 생성  (캐릭터별)
   │
   ▼
메인스토리 자동 요약  (캐릭터+에피소드 → 파생)
   │
   ▼
성과 측정  (에피소드 성과 → 캐릭터 단위 롤업)
   │
   ▼
IP 후보 선정  (전체 캐릭터 성과 랭킹)
```

## 4. 엔티티 모델

| # | 엔티티 | 개수 | 성격 | 소속(FK) |
|---|---|---|---|---|
| 1 | **world** 세계관 | 1개 고정 | 통제변수 (authored) | — (최상위) |
| 2 | **heroine** 여주 | 1개 고정 | 통제변수 (authored) | world (고정) |
| 3 | **character** 남캐 | 다수 | 실험 변수 · 독립 DB (authored) | world (고정) |
| 4 | **episode** 에피소드 | 캐릭터당 3~5 | 시행 (authored) · 스토리보드 내포(후순위) | **character** |
| 5 | **story** 메인스토리 | 캐릭터당 1 | **자동 요약 (파생)** | character |
| 6 | **performance** 성과 | 캐릭터당 1 | **자동 롤업 (파생)** | character |
| 7 | **IP후보** | 전체 1 | **자동 랭킹 (파생, 최종 산출물)** | — |

핵심: **손으로 관리하는 링크는 `episode → character` 하나뿐.**
세계관·여주는 전역 고정 상수라 링크가 아니고, 메인스토리·성과·IP후보는 전부 자동 생성 파생물이라 깨질 링크가 없다.

## 5. 폴더 구조 (캐릭터 중심 종속)

각 남캐 = **폴더 하나 = 자기완결 실험 레코드.** 에피소드가 그 안에 물리적으로 종속된다.

```
기획-v1/
├── 00_규칙엔진/               검증 기준 (그대로 유지)
│    ├── engine.py
│    ├── 규칙_최신.json / .md
│    ├── 로우데이터/
│    └── 발행성과/
├── 01_세계관.json             ← 고정 1개
├── 02_여주.json               ← 고정 1개
├── 03_캐릭터/                 ← 남캐 독립 DB
│    ├── char_민재/
│    │    ├── character.json          (남캐 정의 · authored)
│    │    ├── episodes/
│    │    │    ├── ep_01.json         (시행 · authored)
│    │    │    ├── ep_02.json
│    │    │    └── ep_03.json
│    │    ├── _story.json             (메인스토리 자동요약 · 파생)
│    │    └── _performance.json       (성과 자동롤업 · 파생)
│    ├── char_서준/
│    │    └── …
│    └── …
├── 04_심사위원/               채점 게이트 (그대로)
└── _IP후보.json               전체 캐릭터 성과 랭킹 (자동 · 최종 산출물)
```

**규칙:** `_` 접두사 = 자동 생성 파생물, 손대지 않는다 (`규칙_최신.json`과 같은 철학).

## 6. ID 규칙

- 캐릭터: `char_{영문슬러그}` (예: `char_minjae`). 폴더명과 일치.
- 에피소드: 캐릭터 폴더가 네임스페이스이므로 폴더 안에서 `ep_01`, `ep_02`로 짧게. 전역 고유성은 경로(`char_minjae/ep_01`)로 보장.
- 불투명 랜덤 ID(`char__6za`, `story__oa0`) 금지. 생성 시 슬러그 + 중복 검사.

## 7. 자동 생성 로직 (파생물)

- **_story.json (메인스토리 자동요약):** 캐릭터 정의 + 에피소드 beats를 묶어 한 줄 로그라인 + 아크 요약 생성.
- **_performance.json (성과):** 캐릭터의 모든 에피소드 metrics(views, ER%, 저장률)를 집계 → 캐릭터 단위 점수.
- **_IP후보.json:** 전체 캐릭터를 성과 기준 정렬 → 상위 + 기준 통과분에 IP 후보 태그.
