## 1. 무엇을 왜 만들었나

영상문법가이드 §2·§3의 수치·순위 주장(클로즈업 기본값, 대사 유형 순위, 트로프 목록, 상위권 컷 특성)은 이 저장소의 `00_규칙엔진/규칙_최신.json`에서 왔다. 크롤링 배치가 발행될 때마다 규칙은 자동 재계산되지만 가이드는 수동 개정이라, 둘이 어긋나는(drift) 시점을 사람이 알기 어렵다.

**설계 원칙: 규칙이 가이드를 자동으로 고치는 것은 금지.** 창작 규칙은 안정성이 우선이고 표본 요동에 출렁이면 안 된다. 그래서 어긋남을 **감지해서 사람에게 개정 판단을 요청**하는 구조로 만들었다.

## 2. 구조

```
[크롤러 publish.py] → CSV 커밋 → [recalc-rules.yml]
    ① engine.py --content-type drama_clip  → 규칙_최신.json 갱신
    ② kit 체크아웃(읽기 전용) → guide_drift_check.py 실행
    ③ "가이드 정합성: OK n / DRIFT n / WEAK n" 한 줄을
       Actions summary + 규칙 갱신 커밋 메시지에 기록
       (※ 여기서는 감지·보고만 — kit에 push 불가/안 함)

[kit의 drift-check.yml]  ← 매일 06:00 KST + 가이드/스크립트 변경 push + 수동
    규칙을 이 저장소 raw URL에서 받아 검사 → 가이드/drift_report.md 갱신 커밋
```

- 이원화 이유: Actions `GITHUB_TOKEN`은 타 저장소 push 불가. 그래서 리포트 파일 커밋은 kit 자신의 워크플로가 하고, 이 저장소는 요약 한 줄만 남긴다. (PAT 시크릿을 쓰면 즉시 반영형으로 바꿀 수 있으나 현행 주 1회 크롤링 주기엔 일일 스케줄로 충분)

## 3. 앵커(rule_anchor) — 가이드 쪽 계약

가이드(`drama-production-kit/가이드/영상문법가이드_v10.1.md`)의 주장 옆에 HTML 주석으로 심어져 있다. JSON 한 줄:

```
<!-- rule_anchor: {"claim_id":"closeup_default","axis":"cut_metrics","field":"median_closeup_ratio","relation":">=","threshold":0.8,"tier":"min","guide_text":"클로즈업/상반신이 dominant shot(기본값)"} -->
```

현재 5개: `closeup_default`(§2-1) · `top_faster_cuts`(§2-3) · `dialogue_top_confrontation`(§2-4) · `save_share_tropes`(§2-5) · `top_first3s_closeup`(§3).

relation 종류: `>=`/`<=`/`>`/`<`(cut_metrics 수치, `tier`: top_third/bottom_third/min), `top_gt_bottom`(상·하위 비교), `in_top`(랭킹 축에서 tags가 상위 threshold위 안 — `low_sample_policy:"exclude"`면 low_sample 항목은 순위에서 제외).

판정: **OK**(일치) / **DRIFT**(수치·순위 어긋남) / **WEAK**(근거 태그가 low_sample로 떨어짐). DRIFT·WEAK ≥1이면 리포트 맨 위에 "⚠️ 가이드 v10.2 개정 검토 필요" 배너.

**가이드 본문을 개정하면 앵커도 같이 손봐야 한다.** 앵커가 전부 사라지면 스크립트가 에러로 죽는다(의도된 동작 — 계약 파기 감지).

## 4. 현재 판정 (2026-07-05, drama_clip 76편 기준) — 사람이 할 일

`가이드 정합성: OK 3 / DRIFT 2 / WEAK 0` — 상세는 kit의 `가이드/drift_report.md`.

DRIFT 2건은 오탐이 아니라 **content_type 소급(260704) 순위 대변동의 실제 반영**이다:

1. `dialogue_top_confrontation` — "가장 많이 잡힌 대사 유형은 감정추궁/대립" 주장. n=27로 빈도는 여전히 1위지만 **ER 랭킹 10위/14**. 가이드 문구가 "많이 잡힌(빈도)"과 "먹히는(성과)"을 구분 안 하고 있음 → v10.2에서 문구 재작성 검토.
2. `save_share_tropes` — 트로프 목록 중 `forced_choice_or_ultimatum`(선택 강요)이 6위로 밀림(low_sample 제외 순위). 위험/보호·대표님/권력차·오해·배신/복수는 표본 부족(WEAK 태그)이라 근거 무효 상태 → 다음 배치로 표본 확충 후 재판단.

**개정은 자동이 아니다.** 오너가 drift_report.md를 보고 v10.2 개정 여부를 결정할 것.

## 6. 주의사항

- **가이드 본문 자동 수정 금지** — 이 원칙은 kit CLAUDE.md·가이드 머리말·스크립트 docstring에 3중으로 박아뒀다. 어떤 자동화도 `가이드/drift_report.md` 외의 가이드 파일을 쓰면 안 된다.
- 규칙 json 구조(축 이름·필드)를 바꾸면 앵커가 DRIFT("구조 변경?")로 떨어진다. engine.py 산출 스키마 변경 시 kit 앵커도 마이그레이션할 것.
- kit의 drift-check.yml은 `가이드/drift_report.md`만 커밋한다(트리거 paths에 없어 루프 없음).
- 이 저장소 recalc-rules.yml의 drift 단계는 `continue-on-error: true` — 검사가 죽어도 규칙 갱신 커밋은 막지 않는다(그 경우 커밋 메시지에 "검사 실패" 표기).
